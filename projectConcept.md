Here’s a project idea that uses SurrealDB and LangChain to build an agent with structured memory and persistent context that solves a meaningful problem while aligning with the hackathon’s rules and judging criteria.

## Project concept – **“ResearchGraph Assistant”**

**Problem addressed:**
Modern researchers drown in papers and citations. Finding influential work, understanding how ideas connect, and recalling insights across sessions is difficult. Traditional RAG systems rely on flat vector search, which often returns disjointed chunks of text and misses relational context. Knowledge graphs, however, enable multi‑hop reasoning, deterministic accuracy and explainability. The hackathon challenges teams to build agentic workflows grounded in structured memory, so this project leverages SurrealDB’s multi‑model capabilities to ingest, organize and query academic literature.

**Solution:**
The ResearchGraph Assistant ingests academic papers (PDFs, arXiv feeds, etc.), uses a large‑language model to extract metadata (title, authors, institutions, topics, key findings, citations) and stores them in SurrealDB as both vector embeddings and graph nodes/edges. Papers become nodes linked to authors and topics; citations are edges pointing to referenced papers; timestamps allow time‑travel queries. Using LangChain/LangGraph, the assistant orchestrates multi‑step workflows: it performs vector searches to find semantically similar papers, traverses the graph to explore citation chains or find influential authors, combines both retrieval methods to build context, and summarises insights. A persistent state is maintained so a researcher can resume sessions later and refine their exploration.

### Key components

| Component                                | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| ---------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Data ingestion & graph creation**      | A LangChain document loader ingests PDFs. An LLM extracts entities (paper, author, topic, date, citations). Using SurrealDB’s multi‑model API, the system stores paper texts in a SurrealDBVectorStore and creates graph records: `Paper` nodes with `authored_by` and `cites` relationships, `Author` nodes linked to institutions, and `Topic` nodes. This converts unstructured text into structured data, similar to the SurrealDB tutorial on extracting structured records from natural language.     |
| **Agentic workflow using LangGraph**     | A LangGraph state machine manages the research workflow: (1) receive a user query; (2) perform semantic search in the vector store; (3) run SurrealQL graph queries to trace citation chains, find related authors/topics or look up evolution over time; (4) ask an LLM to generate a readable summary combining these contexts. Each step is clear, observable and uses LangChain’s standard interfaces.                                                                                                  |
| **Persistent state & versioned context** | SurrealDB supports time‑travel queries and versioned records. The assistant stores session data, user preferences and previously examined papers, allowing researchers to pick up where they left off. Checkpointing and resumable flows are implemented in LangGraph to handle interruptions and multi‑step logic.                                                                                                                                                                                         |
| **Open‑source LangChain integration**    | The hackathon gives bonus points for integrations published as open‑source. This project will contribute at least two reusable components: (1) a `SurrealDBPaperLoader` that parses academic PDF/JSON/BibTeX files and populates SurrealDB; (2) a `SurrealDBCitationRetriever` that exposes a standard LangChain retriever interface to traverse citation graphs. Following the LangChain integration guide (eligibility, implementation, testing and docs) will ensure these contributions are recognised. |
| **User experience**                      | A simple UI (web or CLI) lets a researcher ask questions like “What are the key papers connecting reinforcement learning and neuro-symbolic reasoning since 2020?” The agent returns a ranked list of papers, summaries, and a citation graph path. Persistent notes and reading lists are stored in SurrealDB for later retrieval. Observability is provided through LangSmith traces, satisfying the 10 % “observability” criterion.                                                                      |

### Alignment with hackathon requirements

* **Stack usage:** The challenge requires using LangChain/LangGraph for orchestration and SurrealDB as the persistent data layer. ResearchGraph Assistant relies on SurrealDB for graph, vector and time‑series storage and uses LangChain for agentic workflows.
* **Structured memory/knowledge (30 %)** – SurrealDB’s multi‑model engine stores relational, document and vector data together. The assistant builds a knowledge graph of papers, authors and citations; context evolves as new papers are ingested. Multi‑hop queries enable reasoning across papers.
* **Agent workflow quality (20 %)** – LangGraph orchestrates a clear sequence of tools: semantic search, graph traversal, LLM summarisation and user feedback. The workflow demonstrates tool coordination and reasoning rather than a single call.
* **Persistent agent state (20 %)** – Session information, user preferences and checkpoints are stored in SurrealDB. Time‑travel queries allow the agent to replay past states, satisfying the need for persistent or evolving context.
* **Practical use case (20 %)** – Academic research spans many fields; easing literature review and citation exploration addresses a real‑world pain point. The agent helps users discover influential works, follow citation networks and organise notes.
* **Observability (10 %)** – Using LangSmith for tracing and logging ensures that the agent’s execution is transparent to judges and users. Graph queries and SurrealQL statements can also be displayed for explainability.
* **Bonus – open source** – Publishing the SurrealDB loaders and retrievers as standalone packages (with docs and tests) aligns with the hackathon’s bonus incentive and the LangChain integration guide.

### How this idea leverages SurrealDB & LangChain

* **Unified memory:** SurrealDB’s one‑engine design combines durable storage and fast, agent‑friendly memory, eliminating multiple specialised stores and reducing latency.
* **Graph + vector synergy:** The assistant uses hybrid retrieval – vector similarity to find semantically relevant papers and graph traversal to explore citations – mirroring the hybrid patterns suggested in the hackathon description.
* **Explainable reasoning:** SurrealDB’s knowledge graphs enable deterministic, multi‑hop reasoning and provide clear query paths; this helps reduce hallucinations and makes the assistant’s answers verifiable.
* **Real‑time and temporal queries:** SurrealDB supports real‑time updates and versioned records, allowing the agent to incorporate newly published papers quickly and to examine how ideas have evolved over time.

This project goes beyond a toy demo by delivering a functional research assistant that integrates structured knowledge, persistent memory and agentic reasoning. It also creates reusable components for the LangChain ecosystem, increasing its value beyond the hackathon.
